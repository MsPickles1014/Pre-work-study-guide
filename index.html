Css

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Cascading Style Sheets (CSS), -a programming language used by developers to define how webpage content is presented to users.
  • CSS defines the colors, aesthetics, visual layout, and organization

***Three Ways To Style A Webpage Using CSS***

  • Inline CSS-used to style an element directly in the HTML file
  • Internal CSS style sheet -entire stylesheet embedded directly in the HTML a single change to the CSS rule will apply to all tagged page elements.
  • External CSS style sheet -is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • Selector defines the element or attributes to which the rules, or declarations, will apply.
  • {Declaration} contain two important components: the CSS property we want to apply and the value of the property.
  • The {margin} property indicates how much space we want around the outside of an element
  • The {padding}property adds space around the content inside an element. **Increasing the padding, makes the entire element bigger!**
  • Every HTML element is essentially a box. Each box consists of margins outside the element, and the element's borders, padding, and content.
  • {* selector}- applies a rule to all the elements visible on the page. ** two or more elements that will share the same styling can be grouped together.**
  • {text-align} - is one way to center a title or heading on a webpage.
  • {img} selector assigns property values to any image element in the markup
  • {display} property is assigned with the {block} value, which assigns each image to start on a new line.
  • {link} element creates the connection to the external file.
  • {rel} attribute specifies the relationship between the current document and the linked document resource.
  • {href} attribute specifies the location (URL) of the external resource. **The {href} attribute is using a relative path. This means the location is derived from the current location**
  • {Class} attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector

GIT

  • Git: a version control ststem that lets the developer manage and keep track of each version of a project
  • Github: A cloud based hosting service that allows tracking of projects inside repositories stored remotely
  • It allows developers to colaborate with other developers

    • Basics about Github

  • Allows multiple persons to make changes to project at same time
  • Every project is called a repository
  • Think of a repository a folder that holds all the files to a specific application
  • git status: checks what branch we are currently on

    • Github Workflow

  • Create a branch
  • git checkout -b branch-name: creates a new branch and switches to it
  • git status: confirms you are on the new branch
  • git add -A: stages the file by indiacting to git that we want to save all the changes to the file
  • git commit -m {commit message}
  • git pull origin main: pulls source from github to ensure all changes were made
      • ***if current branch is up to date, message will read: Already up to date***
  • git push origin feature/starter-code: pushes code to individual branch and not main branch
      • ***This is essential to keeping bugs from affecting the main project***
  • git checkout main: directs user back to the main branch
  • git pull origin main: pulls down the lastest version of the main branch
  • git restore {name of file}: restores file to previous state without with out saving changes

Steps To Save Code To Github

  • Step 1. Confirm you are on the correct branch from your command line {git status}
  • Step 2. Use the {git add -A} command in your terminal to stage/commit all your files in the working branch to your working directory staging area of git
  • Step 3. Commit files to your repository with the following command: {git commit -m "added code to HTML file" use the -m flag to associate a message with our commit. It should be short and discriptive.
  • Step 4. check that your current branch is up to date before opening a pull request. {git pull origin main} message should read {Already up to date}
  • Step 5. push up all of our changes to github by using the following command git push origin feature/add-html
  • Step 6. navigate to your feature branch in the repository on GitHub
  • Step 7. click the "Pull Requests" tab at the top of the page
  • Step 8. On the pull requests page, find and click the "New pull request" button
  • Step 9 Compare the two branches that we want to merge..Arrow should be pointing from Compare{Feature} branch to Base{Main} branch.
  • Step 10. Once on the pull request form, give your PR a title and a short description of the changes
  • Step 11. Click the "Create pull request" button.
  • Step 12. Click the "Merge pull request" button and confirm that the message reads { This branch has no conflict with the main branch}.
  • Step 13. A successful merge should read {Pull request succefully merged and closed}
  • Step 14. Navigate back to the main branch in GitHub, and you should see that your merged file is now there
  • Step 15. Update local repository with the new merged-in feature by opening up terminal and entering commad {git checkout main} then {git pull origin main}
  • Step 16. Go back to Github and close the issue associated with this pull request
