HTML

  • Html is used to build a sturdy framework
  • Elements can be text, images, header, or list
  • Element Types: Meta-Related, Content Division and Semantic

      • ***Semantic Element describes its content and the role it plays on the page***
      ***It also makes html easier to be read by other developers and browsers***

  • Each element performs a unique role
  • Head Element contains information about the webpage. Also, contains information or METADATA that helps the browser render the page correcly
  • Body Element: represents the visible content shown to the user.
  • BODY element: contains content that the end user will see in the browser
  • !Doctype HTML: tells the browser the document type it should expect to open
  • HTML tag represents the start or root of html document. Inside the tag is the an attribute that defines additional info about the element.

      • ***html elements are case senative***
      ***html attributes are always defined in the begining element, never the closing***

    Head of HTML Document

  • TITLE: defintions the webpage title (req. element and needed for SEO )
  • SEO-Search Engine Optimization (needed to improve website rank among searches like google, bing, ect)
  • STYLE:defines the Css styles associated with the website
  • BASE: Defines the base URL for webpage
  • LINK: Connects an external resource to the html document
  • META: Defines metadata such as the character set, description, keywords, author and viewpoint
  • SCRIPT: Embeds client-side script in the html document
  • CHARSET: defines the character set (this attribute is defined by Meta Element)
  • /HEAD: Closing tag

      • **All tags are begins with open angle brackets and ends with***

  • Elements can be a parent or a child
  • Parent elements hold nested elementsknown as child elements
  • HEADING ELEMENTS (h1, h2, h3, h4, ect): represents the level of given text box

    • Body of HTML Document

  • Body elements: contain the content that user see when looking at the the webpage in the browser, define the body of the document, and is a semantic HTML element

GIT

  • Git: a version control ststem that lets the developer manage and keep track of each version of a project
  • Github: A cloud based hosting service that allows tracking of projects inside repositories stored remotely
  • It allows developers to colaborate with other developers

    • Basics about Github

  • Allows multiple persons to make changes to project at same time
  • Every project is called a repository
  • Think of a repository a folder that holds all the files to a specific application
  • git status: checks what branch we are currently on

    • Github Workflow

  • Create a branch
  • git checkout -b branch-name: creates a new branch and switches to it
  • git status: confirms you are on the new branch
  • git add -A: stages the file by indiacting to git that we want to save all the changes to the file
  • git commit -m {commit message}
  • git pull origin main: pulls source from github to ensure all changes were made
      • ***if current branch is up to date, message will read: Already up to date***
  • git push origin feature/starter-code: pushes code to individual branch and not main branch
      • ***This is essential to keeping bugs from affecting the main project***
  • git checkout main: directs user back to the main branch
  • git pull origin main: pulls down the lastest version of the main branch
  • git restore {name of file}: restores file to previous state without with out saving changes
